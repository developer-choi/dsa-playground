## 메타 원칙 — "정렬 후 인덱스가 무엇을 의미하는가?"

정렬 문제를 만나면 가장 먼저 이 질문을 던진다.

> 정렬한 배열에서 **원소의 위치(인덱스) 자체가 답이 되는가?**

답이면 누적합도, frequency map도, 부가 자료구조도 필요 없다. 정렬 한 번 + 인덱스 읽기로 끝.

| 인덱스의 의미 | 적용 |
|---|---|
| 앞에 몇 개 있는가 | 등수, 백분위, "나보다 큰/작은 수 카운트" |
| 중앙 위치 | 중앙값, K번째 원소 |
| 두 배열의 교차 위치 | 정렬 배열 병합·교집합 |
| 압축된 좌표 | 좌표 압축 |

인덱스에 답이 안 박혀있을 때만 frequency map·누적합 같은 **두 번째 도구**를 꺼낸다. 순서 거꾸로 가면 자료구조가 부풀고 산식이 우회한다.

## 동점자 ranking — 정렬된 배열의 "첫 등장 인덱스" = 등수 - 1

### 사고 흐름

**문제 정의**: 각 원소에 대해 "나보다 큰 값의 개수 + 1" 을 구한다.

**naive 기각**: 각 원소마다 전체 훑기 = O(N²). 입력이 크면 터짐.

**핵심 관찰** — 각 원소의 답은 **독립이 아니다**.
점수 5의 "나보다 큰 수"가 X라면, 점수 4의 "나보다 큰 수"는 X + (점수 5인 사람 수).
즉 **큰 쪽부터** 차례로 보면 답이 **누적으로 쌓인다**.

**도구 호출**: "큰 쪽부터 본다" → **내림차순 정렬**.
(누적 방향이 답의 방향과 일치해야 한다. 답이 "큰 쪽 카운트"인데 오름차순으로 가면 매번 보수 산식 `전체 - acc - count`로 뒤집어야 함 — 그 산식 등장이 **방향 잘못 잡았다는 신호**.)

**답 읽기**: 내림차순 정렬된 배열에서 어떤 값의 **첫 등장 인덱스 i** = 그 값보다 큰 사람 수 = **등수 - 1**.

**동점 처리**: 동점자는 정렬된 배열에서 자연스럽게 인접 → "첫 등장 인덱스만 기록"으로 한 줄 처리. **별도 그룹화(frequency map) 불필요** — 그룹화는 정렬의 부산물이지 명시적 도구가 아니다.

**원본 순서 복원**: `Map<값, 등수>` 만들어두고 마지막에 `input.map(v => map.get(v))`. 결과값이 입력값의 함수일 때(같은 값 → 같은 결과) 일반화 가능 — 등수·백분위·빈도순위·좌표 압축이 모두 이 형태.

### 갈림길 — frequency map으로 도망가지 마라

핵심 관찰 단계에서 "동점자 그룹별 처리" 쪽으로 점프하면 frequency map부터 만들게 된다. 그 순간 정렬은 "그룹화의 부산물"이 되고, 답은 그룹 단위 누적 산식으로 우회된다 (자료구조 2개, 루프 3개, mutation 발생).

frequency map은 **"정렬해도 인덱스에 답이 안 보일 때"** 꺼내는 두 번째 도구다. 이번 문제처럼 인덱스 자체가 답이면 첫 번째 도구만으로 끝.

### 코드

```ts
const sorted = [...grade].sort((a, b) => b - a);
const rankByGrade = new Map<number, number>();
for (let i = 0; i < sorted.length; i++) {
  if (!rankByGrade.has(sorted[i])) rankByGrade.set(sorted[i], i + 1);
}
return grade.map((g) => rankByGrade.get(g)!);
```
