# 문제 독해 체크리스트

AI 리뷰어가 힌트·꼬리질문을 설계할 때 참조하는 레퍼런스. 사용자는 훈련을 방해받지 않도록 직접 보지 않는다 — 막힐 때 리뷰어가 단계별 힌트로 풀어주고, 접근법 제출 시 꼬리질문으로 검증한다.

## 3가지 신호

### 1. 문제의 "형태 패턴" 식별

| 패턴 | 예시 언어 | 도구 후보 |
|------|----------|----------|
| "각 원소마다 ~한 **첫 번째** 원소" | "오른쪽에서 더 큰 첫 값" | 단조 스택 |
| "**그룹별로** 집계" | "장르별", "단어별" | 해시맵 |
| "**상위/하위 K개**" | "가장 큰 3개" | 힙 |
| "**연속 구간**의 최대/최소/합" | "길이 k짜리 구간 합 최대" | 슬라이딩 윈도우 |
| "정렬된 배열에서 **조건 만족하는 값**" | "k 이하 최대값" | 이분탐색 |
| "**짝/짝짓기**" | "합이 k 되는 두 수" | 해시맵 / 투포인터 |
| "**최적값 + 중복 부분문제**" | "최대 부분 수열 합" | DP |
| "**문제를 반으로 쪼갤 수 있음**" | 분할-정복 가능 구조 | 재귀/분할정복 |
| "**상태 공간 탐색 + 가지치기**" | "N-Queens" | 백트래킹 |
| "**매 스텝 최적 선택이 전체 최적**" | "동전 교환" | 그리디 |

### 2. 현재 원소의 역할

- **과거 저장본의 운명을 확정** → 스택
- **자기 라벨에 누적** → 해시맵
- **전체 랭킹 Top-K 갱신** → 힙
- **구간 경계 이동** → 투포인터 / 슬라이딩 윈도우

### 3. 저장 규칙의 불변식

한 문장으로 써본다: **"남아있는 애들은 ~한 상태이다."**

예) pg-42584 단조 스택 → "스택에 남은 가격들은 아직 미래에 자기보다 낮은 값이 안 나타난 친구들"

불변식이 곧 저장 규칙의 설계도. 단조성·정렬 상태는 불변식 유지의 부산물.

## 막힘 프로토콜 — 힌트 제공 순서

1. 형태 패턴 힌트 — "어떤 원형?"
2. 현재 원소 역할 힌트 — "들어오는 애가 뭘 하는가?"
3. 불변식 힌트 — "남아있는 애들의 상태를 한 문장으로?"

답은 직접 말하지 않는다. 질문으로만 유도.

## 접근법 제출 시 꼬리질문 순서

사용자 사고 경로를 드러내는 것이 목적 (정답 검증 아님).

1. **"왜 그 도구를 선택했나요?"** → 직감/분석 분기
   - 직감: "처음 보는 유사 문제에서도 떠올릴 수 있을까요?" → 체크리스트로 재구성
   - 분석: "어느 신호에서 포착했죠?" → 형태 패턴/역할/불변식 단계 확인
2. 저장 규칙: "**뭘** 저장? pop/push/lookup **기준**?"
3. 불변식: "남아있는 애들의 상태를 **한 문장**으로?"
4. 부산물 인식: "단조성·정렬 상태, **목표**인가 **결과**인가?"

톤: 면접관처럼 차갑게. `/discussion` 스킬 참고.

## 도구별 "핵심 활용" 형식

회고 시 "핵심 활용" 칸은 도구에 따라 형식이 달라진다. 도구 이름만 적으면 복습에 무용하니 아래 형식을 반드시 채운다.

| 도구 | 핵심 활용의 형식 |
|------|----------------|
| 자료구조 (스택, 힙, 해시…) | 불변식: "남아있는 애들은 ~한 상태" |
| 재귀 / 분할정복 | 기저 조건 + 재귀식 |
| DP | 상태 정의 + 점화식 |
| 이분탐색 | 탐색 대상 + 조건 함수 |
| 투포인터 / 슬라이딩 윈도우 | 두 포인터의 의미 + 이동 조건 |
| 백트래킹 | 상태 공간 + 가지치기 조건 |
| 그리디 | 선택 기준 + 왜 최적인지 (논리) |
