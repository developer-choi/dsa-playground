# 완전탐색 vs DP

학습 기반:
- [pg-43165 타겟 넘버](../src/problems/dfs-bfs/pg-43165.ts) — 경우의 수 세기
- [pg-42629 라면공장](../src/problems/heap/pg-42629.ts) — 최소 공급 횟수

두 도구 모두 "모든 경우를 본다". 차이는 **중복된 중간 계산을 재사용하는가**.

## 완전탐색

각 경로를 처음부터 끝까지 **중복된 중간 계산을 매번 다시** 한다. N이 작을 때(대략 ≤20~25)만 현실적. 재귀로 구현되곤 한다.

## DP

같은 중간 상태를 만나면 **공책(표)에 적어둔 답을 재사용**한다. 원리적으로는 완전탐색의 상위 호환 — 공책이 쓸모없으면(중복 없음) 완전탐색과 동일하고, 쓸모있으면 극적으로 빨라진다. 단 **공책 크기(상태 수)가 감당 가능해야** 실전 풀이가 된다.

## DP는 완전탐색을 어떻게 줄이나 — 같은 상황 합치기 + 손해 갈래 버리기

DP는 완전탐색과 **다른 알고리즘이 아니다**. 완전탐색이 그리는 2^N 갈래에서 군더더기를 쳐낸 것일 뿐. 쳐내는 방법은 둘:

- **같은 상황 합치기**: 서로 다른 선택으로 도달했어도 "지금 상태"가 같으면 이후 미래가 동일 → 한 번만 계산하고 재사용.
- **손해 갈래 버리기**: 같은 상태인데 더 나쁜 쪽은 미래도 무조건 나쁨 → 그 갈래를 통째로 버림.

예) 공급 후보 `[5, 3]`, 받기/안받기 완전탐색 = 4갈래:

| 선택 | 받은 횟수 | 재고 |
|---|---|---|
| 안받고 안받고 | 0 | +0 |
| 안받고 받고 | 1 | +3 |
| 받고 안받고 | 1 | +5 |
| 받고 받고 | 2 | +8 |

"받은 횟수 1"이 두 개(+3, +5)다. **+5가 무조건 이득**(앞으로 더 버팀)이라 +3 갈래는 버린다. 4갈래가 "횟수별 최선" 3개로 압축됨 → 이게 `dp[받은 횟수] = 최대 재고`. (완전탐색은 2^N을 끝까지 다 들고 가고, DP는 이렇게 중간에 버려 갈래 수를 줄인다.)

## 먼저 물을 것 — "가짓수냐, 최소·최대냐"가 압축 가능 여부를 가른다

위처럼 갈래를 **버릴 수 있느냐**가 DP와 완전탐색을 가른다. 그 기준은 구하는 값의 종류:

| 구하는 것 | 손해 갈래 버리기 | 결론 |
|---|---|---|
| **가짓수** (몇 가지냐) | ❌ 모든 잎이 답에 1씩 보탬 → 하나도 못 버림 | 완전탐색 |
| **최소 / 최대** 하나 | ⭕ 최선만 필요 → 손해 갈래 버림 | DP·그리디 가능 |

pg-43165(타겟넘버)는 **가짓수**라 어떤 경로도 못 버려 완전탐색. pg-42629(라면공장)는 **최소 횟수**라 손해 갈래를 버릴 수 있어 DP·그리디가 열린다.

## 떠올리는 순서

1. 부분 상태를 정의할 수 있는가. 같은 상태가 여러 경로로 도달하는가 → 중복 있음 → **DP 후보**.
2. 상태의 총 개수가 관리 가능한가(대략 수백만 이하) → **DP 성립**.
3. 중복이 없거나 상태 수가 폭발 → **완전탐색** (N 작은지 확인).

## 두 문제에 적용한 결과

| 구분 | pg-43165 타겟넘버 | pg-42629 밀가루 |
|---|---|---|
| N 제약 | ≤ 20 | ≤ 20,000 |
| 완전탐색 | ⭕ 정답 (= 재귀) | ❌ TLE |
| DP | ⭕ 대안 풀이 성립 (쓰지 않음) | ❌ 상태 폭발로 실전 불가 |

pg-43165에서 DP를 **쓰지 않은 이유**: N이 작아 완전탐색(2^20 ≈ 100만 연산)만으로 충분히 통과하기 때문에, 상태 표(`dp[i][current_sum]`)를 관리하는 추가 복잡도를 감수할 이유가 없다. DP로도 풀리지만 실전 풀이 우선순위에서 완전탐색에 밀린다. "DP가 성립하지만 쓸 필요가 없는 경우"의 사례 — 완전탐색이 이미 통과 가능한 범위라면 더 단순한 쪽이 낫다.
