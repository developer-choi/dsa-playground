# 완전탐색 vs DP

학습 기반:
- [pg-43165 타겟 넘버](../src/problems/dfs-bfs/pg-43165.ts) — 경우의 수 세기
- [pg-42629 라면공장](../src/problems/heap/pg-42629.ts) — 최소 공급 횟수

두 도구 모두 "모든 경우를 본다". 차이는 **중복된 중간 계산을 재사용하는가**.

## 완전탐색

각 경로를 처음부터 끝까지 **중복된 중간 계산을 매번 다시** 한다. N이 작을 때(대략 ≤20~25)만 현실적. 재귀로 구현되곤 한다.

## DP

같은 중간 상태를 만나면 **공책(표)에 적어둔 답을 재사용**한다. 원리적으로는 완전탐색의 상위 호환 — 공책이 쓸모없으면(중복 없음) 완전탐색과 동일하고, 쓸모있으면 극적으로 빨라진다. 단 **공책 크기(상태 수)가 감당 가능해야** 실전 풀이가 된다.

## 떠올리는 순서

1. 부분 상태를 정의할 수 있는가. 같은 상태가 여러 경로로 도달하는가 → 중복 있음 → **DP 후보**.
2. 상태의 총 개수가 관리 가능한가(대략 수백만 이하) → **DP 성립**.
3. 중복이 없거나 상태 수가 폭발 → **완전탐색** (N 작은지 확인).

## 두 문제에 적용한 결과

| | pg-43165 타겟넘버 | pg-42629 밀가루 |
|---|---|---|
| N 제약 | ≤ 20 | ≤ 20,000 |
| 완전탐색 | ⭕ 정답 (= 재귀) | ❌ TLE |
| DP | ⭕ 대안 풀이 성립 (쓰지 않음) | ❌ 상태 폭발로 실전 불가 |

pg-43165에서 DP를 **쓰지 않은 이유**: N이 작아 완전탐색(2^20 ≈ 100만 연산)만으로 충분히 통과하기 때문에, 상태 표(`dp[i][current_sum]`)를 관리하는 추가 복잡도를 감수할 이유가 없다. DP로도 풀리지만 실전 풀이 우선순위에서 완전탐색에 밀린다. "DP가 성립하지만 쓸 필요가 없는 경우"의 사례 — 완전탐색이 이미 통과 가능한 범위라면 더 단순한 쪽이 낫다.
