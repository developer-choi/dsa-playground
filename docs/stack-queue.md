# Stack / Queue

## Stack

### 언제 쓰나 (형태 패턴)
- "각 원소마다 미래/과거 방향 **첫 번째 조건 맞는 값**" → 단조 스택
- "중첩 / 되돌리기" → LIFO 본질 그 자체

### 사례

#### [pg-42584 주식가격](../src/problems/stack-queue/pg-42584.ts)
- **도구**: 단조 스택
- **문제 신호 (형태 패턴)**: 각 원소마다 "**자기보다** 낮은 **첫 번째** 값"까지의 거리. "첫 번째"가 핵심 — 확정→버림이 성립하게 해 스택이 호출됨. "모든 자기보다 낮은 값"이면 스택 안 씀 (이중 루프).
- **관점 전환**: "각 원소가 자기 왼쪽/오른쪽을 뒤진다"가 아니라 **"새 원소가 과거 미확정들의 답을 한꺼번에 확정시킨다"** 로 시선을 뒤집어야 스택이 보임.
- **핵심 활용 (불변식)**: 스택에 남은 가격들은 **"아직 미래에 자기보다 낮은 값이 안 나타난 친구들"**. 새 가격이 들어올 때 불변식을 깨는 친구(top > 새 가격)를 pop하며 답 확정. 단조 증가(오름차순)는 불변식 유지의 **부산물**이지 목표가 아님.
- **LIFO 필연성 (정지 조건)**: top부터 pop하면 "top ≤ 새 값"에서 **즉시 멈춤** 가능 → 각 원소 최대 1회 pop → **O(N)**. bottom부터 꺼내면 `[3, 4, 5, 4]` 같은 케이스에서 "3 < 4니까 멈춰도 되나? 아니, 위의 5는 확정 대상"이라 멈춤 조건을 못 세움 → **O(N²)**. LIFO는 **정지 조건이 top에서만 성립**하기 때문에 필연.

#### [pg-12909 올바른 괄호](../src/problems/stack-queue/pg-12909.ts)
- **도구**: 스택 (중첩 매칭)
- **문제 신호 (형태 패턴)**: 중첩 매칭 — "가장 최근 열린 괄호가 다음 닫는 괄호와 짝". `((` 쌓인 상태에서 `)`가 오면 **안쪽** `(`부터 매칭 = LIFO 본질 그 자체.
- **핵심 활용 (불변식)**: 스택에 남은 `(`들은 **"아직 짝을 못 찾은 열린 괄호들"**. `)` 들어올 때 top과 매칭해 pop(확정+버림). 끝에 스택이 비어있어야 모든 괄호가 짝을 찾은 것.
- **필연성 vs 충분성**: 단일 괄호 종류면 **카운터(`let count = 0`)로도 충분** (열림 +1, 닫힘 −1, 음수되면 실패). 괄호 종류가 여러 개(`{`, `[`, `(`)가 되는 순간 **종류 정보를 쌓아야 해 스택이 필연**. 이 문제는 스택이 "충분조건"이지 "필연"은 아니다 — 다만 매칭 문제 패턴을 스택으로 일반화해두는 훈련 가치는 있음.
- **일반화 감각**: 매칭/짝짓기 문제를 만나면 먼저 묻기 — "매칭 대상이 한 종류인가, 여러 종류인가?" 한 종류면 카운터로 충분, 여러 종류면 스택 필수.
