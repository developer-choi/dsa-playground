# Hash

## 언제 쓰나 (형태 패턴)
- "**그룹별 집계**" — 키로 묶고 값에 누적
- "**짝짓기 / 존재 여부 확인**" — 키 lookup으로 O(1)
- "**중복 제거**" — 키의 유일성 활용

## 사례

### [pg-42579 베스트앨범](../src/problems/hash/pg-42579.ts)
- **도구**: 해시맵
- **문제 신호 (형태 패턴)**: 그룹별 집계 + 그룹 간·내 정렬
- **핵심 활용 (저장 구조)**: `{장르: {총재생수, [{곡 재생수, 곡 인덱스}...]}}`. 장르 **간** 정렬은 총합으로, 장르 **내** 정렬은 곡별 재생수로 분리 관리.
- **필연성 (축 A — 복잡도)**: 집계 단계에서 "이 장르 이미 저장됐나?" lookup을 N번 수행. 배열이면 O(N) × N = O(N²), 해시맵이면 O(1) × N = O(N). N=10,000이면 100배 차이.
- **일반화 (pg-12909 대비)**: 키가 **임의 문자열 + 다수**면 해시맵 실질 필연. 키가 **단일·유한**이면 카운터·배열로 대체 가능 (pg-12909 괄호 케이스). "키 종류가 가변·다수 + O(1) 접근 필요" = 해시맵 호출 신호.
