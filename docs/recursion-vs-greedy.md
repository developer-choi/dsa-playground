# 재귀 vs 그리디

학습 기반:
- [pg-43165 타겟 넘버](../src/problems/dfs-bfs/pg-43165.ts) — 경우의 수 세기
- [pg-42629 라면공장](../src/problems/heap/pg-42629.ts) — 최소 공급 횟수

두 도구 모두 "각 지점에서 어떤 선택을 할지 결정"한다. 차이는 **몇 갈래를 여는가**.

## 재귀

각 지점에서 **모든 갈래를 다 열어 탐색**한다. 느리지만 답을 놓치지 않는다. 가짓수 세기처럼 모든 경로의 결과를 합쳐야 하는 문제에 필수.

## 그리디

매 지점에서 **1개만 고르고 직진**한다. 돌아가지 않는다. 빠르지만, "매 순간의 최선이 전체의 최선"이라는 성질이 성립할 때만 정답을 보장한다. 성립 안 되면 오답. 가짓수 세기엔 원천적으로 못 쓴다(한 줄기만 타므로).

## 떠올리는 순서

1. 구하는 게 "가짓수"인가 → 그리디는 아예 불가. **재귀**로 모든 경로 집계.
2. 구하는 게 "최소/최대"인가 → 그리디 가능성 탐색.
3. "매 순간 가장 좋아 보이는 걸 고르면 전체도 최선이다"가 머릿속에서 성립하는가 → **그리디**.
4. 성립 안 된다 → **재귀(완전탐색)** 또는 DP.

## 두 문제에 적용한 결과

| 구분 | pg-43165 타겟넘버 | pg-42629 밀가루 |
|---|---|---|
| 성격 | 카운팅 | 최적화 |
| 재귀 | ⭕ 정답 | ⭕ 동작 (TLE) |
| 그리디 | ❌ 카운팅엔 불가 | ⭕ 정답 |
