# DSA Playground

코딩테스트를 대비해 알고리즘과 자료구조를 공부하는 레포입니다. 정답 코드를 외워 쌓아두는 곳이 아니라, **문제를 보고 풀이 도구를 떠올리기까지의 사고 과정**을 훈련하고 기록하는 데 초점을 두고 있습니다.

그래서 이 레포에서 봐주셨으면 하는 것은 결과물 자체보다 그 결과에 이르는 방식입니다. 알고리즘을 "문제별 정답 패턴"으로 외우지 않고 제대로 이해하려 했는지, 그리고 그걸 위해 어떻게 공부했는지를 아래에 풀어 설명드립니다.

---

## 학습 방식

AI를 **면접관 겸 코드 리뷰어**로 두고 문제를 풉니다. 문제 하나를 잡을 때마다 다음을 지키게 합니다.

- **접근법에 직감이 비치면 꼬리질문으로 되짚습니다.** "왜 그 도구를 골랐나요?"에 "보자마자 떠올랐어요" 같은 답이 나오면 암기로 취급하고, 처음부터 다시 추론해 그 도구가 왜 꼭 필요한지 따집니다.

- **풀이에 버그가 나면 수정법을 알려주는 대신, 버그를 드러내는 테스트만 추가합니다.** 정답을 받아 적는 흐름을 막고 스스로 고치게 합니다.

- **풀이가 끝나면 사고 경로를 카테고리별 문서에 한국어로 풀어 적어 쌓습니다.** 체크리스트가 아니라 문장으로 적어, 복습할 때 답이 바로 보이지 않고 여러 문제에 걸친 공통 패턴을 스스로 찾을 수 있게 합니다.

결과적으로 풀이에 드는 시간은 늘어납니다. 그 대신 "아 이건 스택!"이 아니라 "이런 게 필요한데 → 그러려면 이런 성질이 있어야 하고 → 그 성질을 가진 게 LIFO → 스택"으로 도구를 **도출**하는 훈련이 됩니다.

### 외우지 않고 도출하기: 괄호 짝짓기 예시

> **기억:** "이 문제는 괄호 짝짓기니까 스택 쓰면 되겠네."
>
> **도출:** "닫는 괄호가 들어오면 가장 최근에 열린 괄호와 짝을 맞춰야 한다. 그러니 열린 괄호들을 쌓아두었다가, 닫는 괄호가 올 때 가장 최근 것부터 꺼내 맞춘다. 넣은 역순으로 꺼내는 것 = LIFO → 스택이 필요하다."

둘 다 도착점은 스택으로 같습니다. 하지만 처음 보는 변형 문제가 나오면 앞쪽은 막히고 뒤쪽만 통합니다. 이 레포는 뒤쪽처럼 도구를 스스로 도출하는 연습을 쌓는 곳입니다.

## 한 문제를 푸는 순서

무엇을 왜 하려는지 한국어로 먼저 설명하고, 그 설명이 선 다음에 코드로 옮깁니다. 한 문제를 같은 순서로 풀고, 각 단계를 따로따로 기록해 나중에 다시 봐도 그때의 사고 흐름을 그대로 따라갈 수 있습니다.

```
뼈대 + 테스트 케이스  →  접근법 풀어쓰기 + 꼬리질문 통과  →  코드 작성
```

### 1. 뼈대와 테스트 케이스 먼저

문제를 보자마자 코드부터 쓰지 않습니다. 풀이·테스트 파일만 만들어 두고, 경계와 예외 상황을 먼저 나열합니다.

```ts
it('각 가격에서 가격이 떨어지지 않은 기간을 반환한다', () => {
  expect(fn([1, 2, 3, 2, 3])).toEqual([4, 3, 1, 1, 0]);
});
it('오름차순 후 급락하면 [3, 2, 1, 0]을 반환한다', () => {
  expect(fn([2, 3, 4, 1])).toEqual([3, 2, 1, 0]);
});
it('원소가 1개면 [0]을 반환한다', () => {
  expect(fn([1])).toEqual([0]);
});
```

### 2. 접근법을 한국어로 풀어 쓰고 꼬리질문 통과

목적에서 방법으로 이어지는 흐름으로 접근법을 적습니다. AI가 꼬리질문으로 직감을 걸러내고, 통과한 서술만 카테고리별 문서에 쌓입니다.

> 각 가격마다 "자기보다 낮은 첫 번째 미래 가격까지의 거리"를 구해야 한다.
>
> 낮은 가격을 만나면 그 원소의 답이 정해지니, 그 전까지는 답이 정해지지 않은 채로 대기시켜야 한다.
>
> 아직 답이 정해지지 않은 채 대기 중인 원소들을 담아둘 공간이 필요하다.
>
> 새 가격이 들어오면 그보다 높은 과거 원소들의 답이 한꺼번에 정해져 꺼내진다.
>
> 꺼내는 순서는 가장 최근에 들어온 것부터다.
>
> 넣은 역순으로 꺼내야 한다 → LIFO → 스택이 필요하다.

### 3. 코드 작성: 버그는 답 대신 테스트로

접근법이 잡힌 뒤에야 코드를 씁니다. 버그가 나오면 AI는 답을 주지 않고, 그 버그를 재현하는 테스트만 추가합니다. 저는 그 테스트를 통과시키며 스스로 고칩니다.
