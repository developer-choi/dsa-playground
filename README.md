# DSA Playground

코딩테스트 대비 알고리즘·자료구조 학습 레포입니다. 정답 코드를 외우기보다 풀이에 이르는 사고 과정을 훈련하는 데 초점을 두고 있습니다.

---

## 어떻게 공부하고 있나

Claude Code를 **면접관 겸 코드 리뷰어**로 두고 문제를 풉니다. 문제 하나를 잡을 때마다 AI에게 다음을 요청합니다.

- **접근법을 단계별로 풀어 쓰게 하고, 직감이 보이면 꼬리질문으로 되짚습니다.**

- **풀이에 버그가 나면 수정법을 알려주지 말고, 버그를 드러내는 테스트만 추가하게 합니다.**

- **풀이가 끝나면 사고 경로를 카테고리별 문서에 줄글로 쌓게 합니다.**

결과적으로 풀이 시간은 늘어나지만, "아 이 문제는 스택!"이 아니라 "**이런 게 필요한데 → 그걸 하려면 이 성질이 있어야 하고 → 그 성질을 가진 게 LIFO → 스택**"으로 도구를 **도출**하는 훈련이 됩니다.

### 예시 — 괄호 짝짓기 문제에서의 대비

> **기억:** "이 문제는 괄호 짝짓기니까 스택 쓰면 되겠네."
>
> **도출:** "닫는 괄호가 들어오면 '가장 최근에 열린' 괄호와 짝을 맞춰야 한다. 즉, 열린 괄호들을 쌓아두었다가 닫는 괄호가 올 때 가장 최근 것부터 꺼내 매칭해야 한다. 넣은 역순으로 꺼내는 = LIFO → 스택이 필요하다."

둘 다 결론은 같습니다. 하지만 처음 보는 **변형**이 나오는 순간 전자는 막히고 후자만 통합니다.

---

## 풀이 순서

한 문제를 이 순서로 풉니다. 각 단계가 별도 커밋으로 나뉘어, 커밋 기록에서 사고 흐름을 그대로 따라갈 수 있습니다.

**step 1. 뼈대 잡기 + 테스트 케이스 나열**

문제를 보자마자 구현에 들어가지 않습니다. 풀이·테스트 파일만 만들어 두고, 경계·엣지 케이스를 먼저 나열합니다.

```ts
it('각 가격에서 가격이 떨어지지 않은 기간을 반환한다', () => {
  expect(fn([1, 2, 3, 2, 3])).toEqual([4, 3, 1, 1, 0]);
});
it('오름차순 후 급락하면 [3, 2, 1, 0]을 반환한다', () => {
  expect(fn([2, 3, 4, 1])).toEqual([3, 2, 1, 0]);
});
it('원소가 1개면 [0]을 반환한다', () => {
  expect(fn([1])).toEqual([0]);
});
```

**step 2. 접근법을 줄글로 작성 + AI 꼬리질문 통과**

목적에서 방법으로 이어지는 흐름으로 접근법을 씁니다. AI가 꼬리질문으로 직감을 걸러냅니다. 통과한 서술은 카테고리별 문서에 계속 쌓입니다.

> 1. 각 가격마다 "자기보다 낮은 첫 번째 미래 가격까지의 거리"를 구해야 한다.
> 2. 낮은 가격을 만나는 순간 그 원소의 답이 확정된다 → 그 전까지는 "미확정" 상태로 대기시켜야 한다.
> 3. 대기 중인 미확정 원소들을 담아둘 공간이 필요하다.
> 4. 새 가격이 들어오면, 그보다 높은 과거 원소들의 답이 한꺼번에 확정되어 꺼내 버려진다.
> 5. 꺼내는 순서는 "최근에 들어온 것부터" — 넣은 역순으로 꺼내야 즉시 멈춤 조건이 성립한다.
> 6. LIFO → 스택이 필요하다.

**step 3. 풀이 구현**

접근법이 잡힌 뒤에야 코드를 씁니다. 버그가 나오면 AI는 답을 주지 않고, 해당 케이스를 재현하는 테스트만 추가합니다.
