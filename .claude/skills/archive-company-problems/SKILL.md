기업 코딩테스트 문제를 `docs/company-test/` 아래에 md로 문서화하라.

## 모드 결정

문제마다 프로그래머스·백준·LeetCode 등에서 100% 동일 문제를 검색한다.

- **URL-only 모드**: 완전히 동일한 공개 문제 URL을 찾은 경우
- **full md 모드**: 일치하는 공개 문제가 없는 경우

## 포맷 규칙

### URL-only 모드

```markdown
# {문제 제목}

출처: 기업 코딩테스트
참조: {공개 문제 URL}
```

### full md 모드

```markdown
# {문제 제목}

출처: 기업 코딩테스트

## 문제 설명

{문제 설명}

## 제한사항

{제한사항 bullet list}

## 입출력 예

| 입력1 | 입력2 | result |
|-------|-------|--------|
| ...   | ...   | ...    |

### 예제 설명

{예제 설명}

## 참조 문제

- [{플랫폼} - {유사 문제 제목}]({URL})
```

- `출처: 기업 코딩테스트`를 반드시 쓴다. 이미지 브라우저 주소창에 URL이 보여도 공개 URL이 아니므로 출처에 쓰지 않는다.
- `## 참조 문제` 섹션은 유사 공개 문제가 있을 때만 추가한다. 없으면 생략.
- URL-only 모드에서는 문제 설명·제한사항·입출력 예를 쓰지 않는다.

## 파일명

`docs/company-test/problem{N}.md` — `docs/company-test/`를 Glob으로 확인해 기존 최대 번호에 이어서.

## 절차

1. `$ARGUMENTS`로 전달된 이미지를 Read하거나 사용자가 설명한 내용을 파악한다.
2. 문제마다 공개 플랫폼에서 동일 문제를 검색해 모드를 결정한다.
3. `docs/company-test/`를 Glob으로 확인해 다음 번호 N을 결정한다.
4. 문제가 여러 개면 각각 별도 파일로 생성한다.
5. 파일 생성 후 커밋한다. 커밋 메시지: `feat(docs): {문제 제목} 문서 추가`
