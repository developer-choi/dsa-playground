# DSA Playground

> 코딩테스트 대비 알고리즘·자료구조 학습 레포. 단순 정답 코드가 아니라 **문제를 마주한 순간부터 풀이 도구를 호출하기까지의 사고 경로**를 훈련·기록하는 데 초점을 둡니다.

이 레포에서 봐주셨으면 하는 것은 세 가지입니다. 첫째, 커밋 로그에 쌓이는 **꾸준한 학습 이력**. 둘째, 알고리즘·자료구조를 단순히 "문제별 정답 패턴"으로 외우지 않고 **제대로 이해하려는 시도**. 셋째, 그것을 가능하게 만든 **구체적인 공부 방식**. 아래에서 두 번째·세 번째가 실제로 어떻게 드러나는지를 설명드립니다.

---

## 어떻게 공부하고 있나

Claude Code를 **면접관 겸 코드 리뷰어**로 두고 문제를 풉니다. 문제 하나를 잡을 때마다 AI에게 다음을 시킵니다.

- **접근법을 "목적/방법 연쇄 서사"로 쓰게 시키고, 직감이 보이면 꼬리질문으로 파기.** "왜 그 도구를 선택했나요?"에 "보자마자 떠올랐어요" 같은 경험 기반 답이 나오면 지름길(암기)로 취급하고 정석 연쇄 추론으로 재구성하게 합니다. 같은 문제를 다른 도구로 풀 수는 없는지, 복잡도 차이는 얼마인지까지 반례를 들어 **필연성**을 검증합니다.

- **풀이에 버그가 나면 수정법을 알려주지 말고, 버그를 드러내는 테스트만 추가하게.** 정답 코드를 복붙하는 흐름을 의도적으로 차단하고, 스스로 고치게 만듭니다.

- **풀이가 끝나면 사고 경로를 카테고리별 문서에 내러티브로 쌓게.** 카드식 체크리스트가 아니라 문단 서사로 씁니다. 복습 시 답이 바로 보이지 않도록, 그리고 여러 문제를 관통하는 공통 패턴을 스스로 추출할 수 있도록.

결과적으로 풀이 시간은 늘어나지만, "아 이 문제는 스택!"이 아니라 "**이런 게 필요한데 → 그걸 하려면 이 성질이 있어야 하고 → 그 성질을 가진 게 LIFO → 스택**"으로 도구를 **도출**하는 훈련이 됩니다.

### 예시 — 괄호 짝짓기 문제에서의 대비

> **기억:** "이 문제는 괄호 짝짓기니까 스택 쓰면 되겠네."
>
> **도출 ([`docs/stack-queue.md`](./docs/stack-queue.md)):** "닫는 괄호가 들어오면 '가장 최근에 열린' 괄호와 짝을 맞춰야 한다. 즉, 열린 괄호들을 쌓아두었다가 닫는 괄호가 올 때 가장 최근 것부터 꺼내 매칭해야 한다. 넣은 역순으로 꺼내는 = LIFO → 스택이 필요하다. (단일 괄호 종류라면 카운터 하나로도 충분하지만, 여러 종류로 확장되면 종류 정보를 함께 쌓아야 해서 스택이 필연이 된다.)"

둘 다 결론은 같습니다. 하지만 처음 보는 **변형**이 나오는 순간 전자는 막히고 후자만 통합니다. 이 레포는 후자의 언어를 늘리는 훈련장입니다.

## 풀이 순서

한 문제를 이 순서로 풉니다. 각 단계가 별도 커밋으로 끊겨 git log에서 사고 흐름을 그대로 따라갈 수 있습니다.

**step 1. 스캐폴딩 + 테스트 케이스 나열** (`9651211`)

문제를 보고 바로 구현에 들어가지 않습니다. 풀이·테스트 파일만 만들고, 경계/엣지 케이스를 먼저 나열합니다.

```ts
it('각 가격에서 가격이 떨어지지 않은 기간을 반환한다', () => {
  expect(fn([1, 2, 3, 2, 3])).toEqual([4, 3, 1, 1, 0]);
});
it('오름차순 후 급락하면 [3, 2, 1, 0]을 반환한다', () => {
  expect(fn([2, 3, 4, 1])).toEqual([3, 2, 1, 0]);
});
it('원소가 1개면 [0]을 반환한다', () => {
  expect(fn([1])).toEqual([0]);
});
```

**step 2. 접근법을 서사로 작성 + AI 꼬리질문 통과** (`23cfddb`)

목적/방법 연쇄로 접근법을 씁니다. AI가 꼬리질문으로 직감을 걸러냅니다. 통과한 서사는 카테고리별 문서에 누적됩니다.

> 각 가격마다 "자기보다 낮은 첫 번째 미래 가격까지의 거리"를 구해야 한다.
> 낮은 가격을 만나는 순간 그 원소의 답이 확정된다 → 그 전까지는 "미확정" 상태로 대기시켜야 한다.
> 대기 중인 미확정 원소들을 담아둘 공간이 필요하다.
> 새 가격이 들어오면, 그보다 높은 과거 원소들의 답이 한꺼번에 확정되어 꺼내 버려진다.
> 꺼내는 순서는 "최근에 들어온 것부터" — 넣은 역순으로 꺼내야 즉시 멈춤 조건이 성립한다.
> LIFO → 스택이 필요하다.
>
> — [`docs/stack-queue.md`](./docs/stack-queue.md), 프로그래머스 42584 회고

**step 3. 풀이 구현** (`4bd84ff`)

접근법이 잡힌 뒤에 코드를 씁니다. 버그가 나오면 AI는 답을 주지 않고 해당 케이스를 재현하는 테스트만 추가합니다.
