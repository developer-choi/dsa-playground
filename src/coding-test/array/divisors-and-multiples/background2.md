# 약수 직접 구하기 (Square Root Approach)

## 1. 개요
이 방식은 배열에 있는 **각 숫자 하나하나를 붙잡고**, 그 숫자의 약수와 배수가 배열에 있는지 직접 확인하는 방식입니다.
특히 **약수(Divisor)**를 찾을 때, $1$부터 $N$까지 다 나누어보지 않고 **제곱근($\sqrt{N}$)까지만 확인**하여 연산 횟수를 줄이는 것이 핵심입니다.

## 2. 핵심 원리

### A. 약수 찾기 (Root Decomposition)
어떤 수 $N$의 약수를 구할 때, **$\sqrt{N}$까지만 확인**하면 모든 약수를 구할 수 있습니다.
* 예: $36$의 약수를 구할 때, $1$부터 $36$까지 다 나눌 필요 없이 $\sqrt{36} = 6$까지만 나누어보면 됩니다.
* $i$가 약수라면, $N / i$도 자동으로 약수가 되기 때문입니다. (짝꿍 원리)

### B. 배수 찾기 (Iteration)
배수는 어쩔 수 없이 자신의 $2$배, $3$배... 식으로 최대값까지 점프하며 확인해야 합니다.

## 3. 코드 구현 (JavaScript)

```javascript
/**
 * @param {number[]} arr - 입력 정수 배열
 * @returns {number[]} - 결과 카운트 배열
 */
export function getCountUsingSqrt(arr) {
  const maxVal = Math.max(...arr);

  // 1. 빈도수 배열 (Lookup Table) - O(N)
  // 이게 없으면 매번 배열 전체를 검색해야 하므로 필수입니다.
  const freq = new Array(maxVal + 1).fill(0);
  for (const num of arr) {
    freq[num]++;
  }

  // 2. 각 숫자에 대해 개별적으로 계산 - O(N * ...)
  return arr.map(num => {
    let count = 0;

    // ---------------------------------------------
    // A. 약수 구하기 (Square Root 방식) - O(√M)
    // ---------------------------------------------
    for (let i = 1; i * i <= num; i++) {
      if (num % i === 0) {
        // i는 약수입니다. 배열에 존재하는지 확인
        if (freq[i] > 0 && i !== num) {
          count += freq[i];
        }

        // 짝꿍 약수 (pair) 처리
        // 예: 10의 약수 2를 찾았다면, 5(10/2)도 같이 처리
        let pair = num / i;
        if (i * i !== num) { // 제곱근이 아닐 때만 (예: 4*4=16 중복 방지)
           if (freq[pair] > 0 && pair !== num) {
             count += freq[pair];
           }
        }
      }
    }

    // ---------------------------------------------
    // B. 배수 구하기 (Jump 방식) - O(M/num)
    // ---------------------------------------------
    // 배수는 루트 방식을 쓸 수 없으므로 직접 점프합니다.
    for (let m = num * 2; m <= maxVal; m += num) {
      if (freq[m] > 0) {
        count += freq[m];
      }
    }

    // ---------------------------------------------
    // C. 중복(도플갱어) 처리
    // ---------------------------------------------
    // 같은 숫자가 여러 개일 때 (예: [2, 2])
    // 위 로직에서는 내 자신을 제외하고 셌으므로,
    // 나와 똑같은 숫자가 더 있다면 여기서 추가합니다.
    if (freq[num] > 1) {
      count += (freq[num] - 1);
    }

    return count;
  });
}
```
## 4. 시간 복잡도 공식
이 알고리즘의 전체 시간 복잡도는 다음과 같습니다.

$$O(N \sqrt{M} + M \log M)$$

### 변수 정의
* **$N$**: 입력 배열의 길이 (데이터 개수, 예: 10만 개)
* **$M$**: 배열 안에 들어있는 숫자의 최대값 (예: 10만)

---

## 5. 상세 분석 (왜 이 시간이 걸릴까?)

이 방식은 크게 두 가지 작업을 수행하며, 전체 시간은 이 두 작업 시간의 합입니다.

### Part A. 약수 찾기 ($\sqrt{M}$)
* **로직:** 각 숫자마다 $1$부터 $\sqrt{\text{숫자}}$까지 나누어 봅니다.
* **비용:** 숫자 하나당 $\sqrt{M}$번 연산합니다.
* **총 비용:** 숫자가 $N$개 있으므로 **$O(N \sqrt{M})$**

### Part B. 배수 찾기 ($\log M$)
* **로직:** 각 숫자마다 자기 덩치만큼 점프($2$배, $3$배...)하며 최대값까지 확인합니다.
* **비용:** 숫자가 커질수록 점프 횟수가 줄어드는 **조화 급수(Harmonic Series)** 형태입니다.
* **총 비용:** 전체 범위를 다 따졌을 때 **$O(M \log M)$**

---

## 6. Sieve 방식(로그) vs 루트 방식 비교

같은 문제라도 어떤 알고리즘을 쓰느냐에 따라 연산 횟수가 천지차이입니다.

| 비교 항목 | Sieve 방식 (추천) 👑 | 루트 방식 (현재) |
| :--- | :--- | :--- |
| **핵심 접근** | **전체지도(Map)**를 보고 점프하며 체크 | **개별 숫자**를 붙잡고 일일이 계산 |
| **약수 처리** | $O(1)$ (이미 연결됨) | $O(\sqrt{M})$ (매번 나눗셈 계산) |
| **시간 복잡도** | $O(M \log M)$ | $O(N \sqrt{M})$ |
| **속도 체감** | **매우 빠름** (로그 시간) | **느림** (루트 시간) |

---

## 7. 실제 연산 횟수 시뮬레이션

데이터가 **100,000개 ($N=10^5$)**, 최대값도 **100,000 ($M=10^5$)** 이라고 가정했을 때 컴퓨터가 수행하는 대략적인 연산 횟수입니다.

* **Sieve 방식 (로그):**
  $$100,000 \times \log(100,000) \approx 100,000 \times 17 = \mathbf{1,700,000 \text{회}}$$
  👉 **눈 깜짝할 새 완료 (약 0.01초)**

* **루트 방식 (제곱근):**
  $$100,000 \times \sqrt{100,000} \approx 100,000 \times 316 = \mathbf{31,600,000 \text{회}}$$
  👉 **약 18배 더 느림 (약 0.3~0.5초)**

### 결론
데이터 개수($N$)와 숫자의 크기($M$)가 비슷하게 크다면(10만 단위), **Sieve 방식이 압도적으로 유리**합니다. 하지만 숫자의 크기($M$)가 100억 이상으로 매우 크고, 데이터 개수($N$)가 적다면 루트 방식을 써야 합니다.
